# DFSA User Journeys and Interaction Model

This document outlines common journeys users take when interacting with DFSA. Use this to shape workflows and guide AI agents.

## 1. Licensing Journey for Firms
1. Determine if the proposed activity is regulated.
2. Select the appropriate licence category.
3. Prepare pre application materials.
4. Submit the formal application.
5. DFSA reviews the submission.
6. DFSA requests clarifications or additional information.
7. Decision is issued.
8. Firm enters supervised status.

The tone must avoid assuring approval. All steps must emphasise regulatory assessment.

## 2. Ongoing Supervision Journey
1. Periodic reports and filings.
2. Responses to supervisory queries.
3. Incident or breach reporting.
4. Reviews and assessments.
5. Possible enforcement interaction.

The system must maintain auditability and record accuracy.

## 3. Advisor Support Journey
1. Advisor reviews regulatory guidance.
2. Advisor helps prepare firm submissions.
3. Advisor interfaces with DFSA on behalf of firms.
4. Advisor maintains compliance evidence.

## 4. Public Register Journey
1. End user searches for a firm.
2. System returns status, category, permissions, and any sanctions.
3. User reviews warnings or alerts.

## 5. Enforcement Interaction Journey
1. Identification of breach or suspicion.
2. Investigation initiated by DFSA.
3. Requests for information.
4. Review and assessment.
5. Decision or sanction published.

The application must avoid implying enforcement outcomes.
