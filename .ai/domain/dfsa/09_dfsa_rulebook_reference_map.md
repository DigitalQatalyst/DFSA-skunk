# DFSA Rulebook Reference Map

This document provides a high level map of DFSA’s primary rulebook modules. This can guide application features that need rule based logic or referential authority.

## 1. General Module (GEN)
Defines regulatory objectives, definitions, core principles, and application of rules. Often used to interpret obligations.

## 2. Authorisation Module (AUT)
Covers licensing requirements for firms and registration requirements for individuals. Relevant for onboarding, approvals, and controlled function management.

## 3. Conduct of Business Module (COB)
Defines obligations on firms when dealing with clients. Includes disclosure, suitability, communication standards, and treatment of client money.

## 4. Prudential Rules (PRU)
Covers financial resources, capital requirements, liquidity, and risk management. Relevant only to firms with prudential obligations.

## 5. Markets Module (MAR)
Regulates market operations, disclosures, market abuse prevention, reporting, and listing requirements.

## 6. Islamic Finance Rules (IFR)
Defines rules for Shari’a compliant activities and products.

## 7. Anti Money Laundering Module (AML)
Covers AML and CTF obligations including customer due diligence, suspicious activity reporting, and internal controls.

## 8. Collective Investment Rules (CIR)
Defines requirements for funds, fund managers, and custody arrangements.

## 9. Fees Module (FER)
Defines DFSA fee structure for applications, licences, and ongoing supervision.

## Usage within the Application
- Forms and workflows should reference the correct module when describing obligations.
- Guidance text should anchor explanations to the relevant rulebook sections.
- Audit logs should capture rule references for transparency and traceability.
